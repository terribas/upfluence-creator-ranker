# Jr Growth Engineer Practical


## The big Idea

```
You have joined Upfluence as a Jr Growth Engineer. One of our strategic priorities is showcasing
our value to prospects and helping brands discover creators faster and make better partnership
decisions. Your goal is to build a small data-driven project that demonstrates your ability to:
- Use AI coding tools (Cursor, Claude, ChatGPT, Lovable, etc.)
- Work with APIs and external data sources
- Structure, process, and analyze data
- Build / vibe code a simple user-facing experience
- Document your work clearly
- Think beyond implementation and consider scalability, productization, and user value
```
## Assignment

#### Task: Ranking of Top Creators by Industry

```
Using the Upfluence API credentials provided and our API documentation, build a publicly
accessible page that showcases the Top 10 Creators by Industry.
Requirements:
Choose 5 industries relevant to B2C influencer marketing (e.g. Beauty, Fashion, Fitness,Food, Gaming).
For each industry:
```

- Retrieve creator data through the Upfluence API
- Define and justify your ranking methodology
- Use multiple creator signals when building your ranking (not a single metric)
- Display the Top 10 creators
- Show the metrics used to support the ranking
- Provide written insights interesting for brands to take into consideration
- Make it easy for a user to browse industries and compare creators

The final result should be accessible through a public URL.

### Deliverables

**_Public Demo_** : Provide a publicly accessible URL (WP, Netlify, Lovable, etc.).
**_GitHub Repository_** : Include all source code and documentation. Your repository should contain:
- Project architecture overview
- Tech stack and AI tools used (ChatGPT, Claude, Cursor, Lovable, etc.)
- Key prompts used during development
- Ranking methodology and rationale
- Setup instructions
- Assumptions made
- Challenges encountered and how they were solved

## Notes

```
- Clearly document your decisions, assumptions, and tradeoffs.
- AI tools are encouraged and expected.
- We care more about your reasoning and execution than perfect code.
- No fluff. Focus on substance over presentation.
```
